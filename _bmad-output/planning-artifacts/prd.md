# Product Requirements Document - moody-blobs-login

**Author:** Mu
**Date:** 2026-02-26

## Executive Summary

moody-blobs-login 是一个纯前端创意登录页，通过四个拟人化几何角色（橙色半圆、紫色长方体、黑色窄条、黄色圆柱）对用户表单交互实时产生情绪反应，将枯燥的登录流程转化为有温度的沉浸式体验。目标用户为创作者本人，核心驱动力是技术与创意的自我表达。

核心问题：登录表单是用户每天都要经历的摩擦点，千篇一律、毫无生气。本项目用"情绪响应"打破这一惯例——角色不是装饰，而是会"感知"用户行为的生命体。

### What Makes This Special

**核心差异点：情绪响应系统。** 不同于普通动画（单向播放），本项目的角色状态由用户行为驱动：输入时担忧、密码可见时背身偷看、登录成功时开心。角色的眼睛、嘴巴、身体倾斜角度全部参与表达，形成多维度的情绪语言。

**核心洞察：** 当界面元素开始"在乎"用户在做什么，用户就会感受到"页面活了"——这种感受来自情绪的双向流动，而非单纯的视觉华丽。

## Project Classification

- **项目类型：** Web App（SPA，纯前端）
- **领域：** General（个人创意/UI 练手）
- **复杂度：** Low
- **项目上下文：** Greenfield（从零开始）

## Success Criteria

### User Success

- 每一个表单交互状态（idle、输入中、密码可见、登录成功）都能触发对应的角色情绪变化，无遗漏
- 四个角色各自拥有完整的情绪表达系统（眼睛、嘴型 SVG Path 变形、身体倾斜），且个性鲜明：橙色最活泼、黄色灵敏、黑色冷静、紫色情绪最丰富
- 动画过渡自然流畅，无跳变或卡顿感
- 完成后可作为作品集展示项，代码质量和视觉完成度达到对外展示标准

### Business Success

- 个人满意度：自己看着爽，愿意放到作品集
- 可部署为独立页面，能通过 URL 分享给他人查看

### Technical Success

- 技术栈：React + Framer Motion
- 情绪状态机逻辑清晰，状态与 UI 严格解耦
- 组件结构合理，角色动画逻辑可维护
- 桌面端流畅运行（移动端适配为 Post-MVP）

### Measurable Outcomes

- **MVP 核心指标：** 4 种情绪状态全部实现且与交互精准对应，四个角色各有个性化表现
- **完成标志：** 自己演示一遍全流程，每个状态切换都觉得"对了"

## Product Scope

### MVP (Phase 1)

**核心用户旅程支持：** J1（Happy Path）、J2（密码偷看）、J4（开发者测试）

- 四个角色 SVG 静态造型
- 全局鼠标视线追踪（idle 态），各角色瞳孔移动权重差异化（橙色 1.2 > 黄色 1.0 > 黑色 0.9 > 紫色 0.8）
- 输入框聚焦时瞳孔锁定光标，随字符数量向右微移
- 四个角色完整情绪状态机（idle / typing / password-visible / success），个性差异化：
  - 橙色：最活泼，反应幅度最大
  - 紫色：情绪主导者，嘴型变化最丰富（4 种 SVG Path 状态）
  - 黄色：视线灵敏，反应适中
  - 黑色：最冷静，反应克制
  - password-visible 态：角色背身偷看——身体背对密码框，眼睛偷偷往密码方向瞟
- 登录表单：邮箱、密码（含显示/隐藏切换）、记住 30 天、忘记密码、登录按钮、注册链接
- **不含 Google 第三方登录**

### Growth Features (Phase 2)

- 移动端适配与触摸交互
- 登录失败情绪状态（如摇头）
- 视差深度效果（角色层次感）
- 页面加载入场动画

### Vision (Phase 3)

- 角色可配置化（颜色、形状可换肤）
- 抽离为可复用的 React 组件库
- 支持注册页、找回密码页等更多场景

### Risk Mitigation

- **技术风险：** SVG Path 变形 + 身体旋转复合动画性能 → Framer Motion `useAnimationControls` 分层控制，动画参数集中配置便于调优
- **复杂度风险：** "背身偷看"复合动作（旋转 + 眼睛反向偷瞟）→ 拆分为独立动画层，分步实现

## User Journeys

### Journey 1: 首次访问者（Happy Path）

**角色：** 任意访客，第一次打开页面。

打开页面，四个几何角色静静站在左侧。鼠标一移动，八颗眼睛齐刷刷跟着转——"这页面是活的？"访客点击邮箱框，角色们的视线集体锁定到输入框，随着字符一个个敲入，瞳孔缓缓向右漂移，像在认真盯着你写字。切换到密码框，紫色角色身体微微前倾，表情变得担忧。密码输完，点击登录，角色们屏住呼吸……成功！紫色嘴角大幅上扬，橙色也跟着开心起来。

**情绪弧线：** 好奇 → 惊喜 → 沉浸 → 满足

---

### Journey 2: 密码偷看者

**角色：** 输入密码后想确认有没有打错的用户。

密码输了一半，点击右侧眼睛图标。密码变成明文的瞬间——角色们集体做出"不该看但忍不住"的反应：身体转向背对密码框，假装没在看，但眼睛却偷偷往密码方向瞟。那种"我不看我不看……但我好奇"的矛盾肢体语言，让用户忍不住笑出来。再次点击隐藏密码，角色们若无其事地转回来，恢复正常。

**情绪弧线：** 犹豫 → 点击 → 被角色的矛盾感染 → 会心一笑

---

### Journey 3: 作品集观众（面试官/朋友）

**角色：** Mu 分享链接给面试官或朋友，对方第一次打开。

"这是我做的一个登录页，你看看。"对方点开链接，以为就是个普通登录页。鼠标移过去，角色眼睛跟着动——"咦？"随手点了一下邮箱框，角色们齐刷刷盯过来。对方开始故意测试：快速移动鼠标、点密码框、开关眼睛图标……每次都有新反应。"这也太有意思了！"

**情绪弧线：** 平淡预期 → 意外惊喜 → 主动探索 → 印象深刻

---

### Journey 4: Mu（开发者测试视角）

**角色：** Mu 在开发过程中验证情绪状态是否正确触发。

每实现一个状态，Mu 就打开页面手动测试：移鼠标看眼睛跟不跟、点输入框看瞳孔锁没锁、输字符看有没有右移、开密码看背身偷看对不对、点登录看成功动画有没有触发。哪个状态不对劲，立刻回去调参数。直到每一个状态切换都让自己觉得"对了，就是这个感觉"。

**情绪弧线：** 期待 → 验证 → 调整 → 满足

## Web App Specific Requirements

### Technical Architecture

- **渲染方式：** CSR，无 SSR/SSG 需求
- **状态管理：** 本地组件状态，无需 Redux/Zustand
- **路由：** 无需路由，单页面
- **构建工具：** Vite
- **动画方案：** Framer Motion ✓

### Browser & Platform Support

- **目标浏览器：** Chrome、Firefox、Safari 最新版
- **目标设备：** 桌面端优先，固定宽度布局可接受
- **移动端：** Post-MVP

### Implementation Considerations

- SVG 角色造型内联于 React 组件，便于动态控制 Path 属性
- 情绪状态机以 React state/context 管理，与渲染层解耦
- 动画参数（时长、弹性系数、偏移量、缩放比例）集中于单一配置文件

## Functional Requirements

### 视线追踪系统

- FR1: 用户移动鼠标时，四个角色的瞳孔跟随鼠标位置偏移
- FR2: 各角色瞳孔移动幅度按权重差异化（橙色 1.2 > 黄色 1.0 > 黑色 0.9 > 紫色 0.8）
- FR3: 用户聚焦输入框时，角色瞳孔从鼠标追踪模式平滑切换至锁定输入框光标位置
- FR4: 用户在输入框中输入字符时，角色瞳孔随字符数量增加向右微移
- FR5: 用户失焦输入框时，角色瞳孔恢复鼠标追踪模式

### 情绪状态系统

- FR6: 系统维护全局情绪状态机，状态：idle / typing / password-visible / success
- FR7: 页面加载完成且无用户操作时，四个角色呈现 idle 状态
- FR8: 用户在任意输入框输入内容时，四个角色切换至 typing 状态
- FR9: 用户点击密码显示/隐藏按钮使密码变为明文时，四个角色切换至 password-visible 状态
- FR10: password-visible 状态下，角色呈现"背身偷看"肢体语言（身体背对密码框，眼睛偷瞟）
- FR11: 用户点击登录按钮且验证通过时，四个角色切换至 success 状态
- FR12: 各情绪状态之间的切换带有平滑过渡动画（200ms～300ms）

### 角色个性系统

- FR13: 橙色角色在所有情绪状态下反应幅度最大（最活泼）
- FR14: 紫色角色为情绪主导者，嘴型变化最丰富（4 种 SVG Path 状态）
- FR15: 黄色角色视线灵敏，反应适中
- FR16: 黑色角色反应最克制（最冷静）
- FR17: 各角色的嘴型通过 SVG Path 变形表达情绪
- FR18: 各角色的身体可发生倾斜/旋转以表达肢体语言

### 角色表情规格（各状态 × 各角色）

| 状态 | 橙色半圆 | 紫色长方体 | 黑色窄条 | 黄色圆柱 |
|------|---------|-----------|---------|---------|
| **idle** | 微笑嘴型，瞳孔正常 | 水平嘴 `M10,20 Q15,18 20,20`，瞳孔正常 | 直线嘴，瞳孔正常 | 点状嘴，瞳孔正常 |
| **typing** | 嘴角下垂（担忧），瞳孔正常 | 下弯嘴 `M40,125 Q55,115 70,125`，身体表现弯曲动画 | 瞳孔放大 ×1.2，无明显嘴变化 | 下弯嘴型呈"O"状（惊叹），瞳孔缩小 ×0.85，未强制锁定 |
| **password-visible** | 旋转 -8°，眼睛偷瞟，瞳孔 ×0.85 | 旋转 -10°，波浪嘴 `M10,20 Q12,18 15,20 Q18,22 20,20`，瞳孔 ×0.85 | 旋转 -5°，眼睛偷瞟，瞳孔 ×0.85 | 旋转 -8°，波浪嘴，瞳孔 ×0.85 |
| **success** | 大笑嘴型，眼睛弯弯，瞳孔正常 | 上扬嘴 `M8,18 Q15,25 22,18`，身体挺直，瞳孔正常 | 嘴微微上扬（克制），瞳孔正常 | 嘴上扬，眼睛弯弯，瞳孔正常 |

**动画参数参考：**
- 瞳孔尺寸：`scale`，spring（stiffness: 200, damping: 20）
- 身体偏转：`rotate`，spring（stiffness: 150, damping: 15）
- 嘴型变形：SVG `d` morphing，250ms，easeInOut
- 偷看眼睛偏移：向密码框方向，约为最大追踪范围的 60%

### 登录表单

- FR19: 用户可在邮箱输入框中输入邮箱地址
- FR20: 用户可在密码输入框中输入密码（默认隐藏为圆点）
- FR21: 用户可点击密码框右侧图标切换密码明文/密文显示
- FR22: 用户可勾选"记住 30 天"选项
- FR23: 用户可点击"忘记密码"链接
- FR24: 用户可点击登录按钮提交表单
- FR25: 用户可点击注册链接跳转注册

### 表单交互与验证

- FR26: 用户提交表单时，系统对邮箱格式进行基础验证
- FR27: 用户提交表单时，系统对密码非空进行验证
- FR28: 表单验证失败时，角色切换至 typing（担忧）状态

## Non-Functional Requirements

### Performance

- 桌面端动画帧率维持 60fps
- 页面首次可交互时间 < 2s
- 所有动画使用 `transform` / `opacity`，避免触发 layout reflow

### Maintainability

- 情绪状态机逻辑与渲染层解耦，状态变更不直接操作 DOM
- 动画参数集中于单一配置文件，调参无需修改组件
- 角色组件与登录表单组件互不依赖，可独立开发和测试
