# 产品需求文档 (PRD)：拟人化动态登录页

## 1. 文档概述

本文档旨在定义一个具有“情绪反馈”的动态登录页面。通过左侧四个几何角色的视觉交互（视线追踪与表情变换），提升用户在表单填写过程中的趣味性与沉浸感。

---

## 2. 页面布局与视觉组件

### 2.1 角色交互区（左侧）

- **角色构成**：
  - **橙色半圆**：位于最前景，移动幅度最大。
  - **紫色长方体**：核心情绪反馈者，位于背景层。
  - **黑色窄条**：位于中景，保持相对冷静。
  - **黄色圆柱**：位于右侧，视线灵敏。
- **视觉深度**：建议实现简单的视差效果。瞳孔移动权重：橙色 (1.2) > 黄色 (1.0) > 黑色 (0.9) > 紫色 (0.8)。

### 2.2 登录表单区（右侧）

- **标题**：`欢迎回来！`
- **副标题**：`请输入您的详细信息`
- **输入框**：
  - 邮箱 (Email)
  - 密码 (Password)：含右侧“眼睛”图标切换按钮。
- **操作项**：记住 30 天、忘记密码、登录按钮、Google 第三方登录、注册链接。

---

## 3. 核心交互逻辑

### 3.1 全局视线追踪系统 (Eye Tracking)

- **非聚焦态 (Idle)**：所有角色的瞳孔中心点跟随鼠标指针 $(x, y)$ 在一定范围内偏移。
- **聚焦态 (Focus)**：
  - 当点击邮箱或密码框时，瞳孔视线需从当前位置平滑平移（Lerp）并锁定在输入框的光标 (Caret) 位置。
  - **动态位移**：瞳孔位置应随输入的字符长度增加而向右微动，模拟“看着你写”的效果。

### 3.2 紫色角色表情状态机 (Emotion State Machine)

紫色长方体是页面的“情绪中心”，其嘴部（SVG Path）需根据以下逻辑切换：

| 用户状态              | 嘴部形态 (SVG)                                  | 视觉语义      | 触发条件                                 |
| :-------------------- | :---------------------------------------------- | :------------ | :--------------------------------------- |
| **初始/未激活**       | 微笑/水平 (`M10,20 Q15,18 20,20`)               | 平静          | 页面加载完成，无操作。                   |
| **正在输入/点击登录** | **向下弯曲** (`M10,20 Q15,25 20,20`)            | **担心/忧虑** | 正在输入内容、点击登录按钮或校验未通过。 |
| **开启“显示密码”**    | **波浪线** (`M10,20 Q12,18 15,20 Q18,22 20,20`) | **偷看/尴尬** | 点击密码框右侧眼睛图标，密码变为明文。   |
| **验证通过**          | 大幅上扬 (`M8,18 Q15,25 22,18`)                 | 开心          | 登录接口请求成功。                       |

---

## 4. 技术实现要点 (For AI Developer)

### 4.1 动画参数

- **表情过渡**：嘴部 SVG 的 `d` 属性变换需配合 `transition` 或 `framer-motion` 的 `layout` 属性，过渡时间建议 $200ms \sim 300ms$。
- **吸附平滑度**：瞳孔从跟随鼠标到锁定光标的切换应带有弹性（Spring）效果，避免僵硬跳变。

### 4.2 逻辑伪代码示例

```javascript
const getMouthPath = (status) => {
  if (isPasswordVisible) return PATH_WAVY; // 偷看态
  if (isError || isTyping) return PATH_DOWN; // 担心态
  return PATH_SMILE; // 平静态
};
```
